# Changelog

All notable changes to Darwin Chronicles will be documented in this file.

Format follows [Keep a Changelog](https://keepachangelog.com/).

## [Unreleased]

### Added
- Project scaffolding: CMake build system with C++20, FetchContent dependencies
- Six module static libraries: environment, world, organisms, evolution, simulation, rendering
- Stub headers and source files for all planned components
- Catch2 test harness with placeholder tests for each module
- Google Benchmark harness for performance testing
- Example programs: earth_like, mars_like
- Makefile convenience targets (build, test, bench, run, format, clean)
- Compiler warning target (`darwin_warnings`) with aggressive diagnostics
- PixelOdyssey terrain tilesets (16x16, 32x32, 48x48) covering 12 biome themes
- Architecture Decision Records (ADRs 0001–0006)
- Project documentation: ARCHITECTURE.md, CHANGELOG.md, docs README
- System design documents: planetary model, world generation, nutrient regeneration, neural evolution, sprite generation
- Sandbox layer: `sandboxes/` directory with `add_darwin_sandbox` CMake helper; ADR-0007 records the decision (see `docs/decisions/0007-sandbox-layer.md`)
- `sandboxes/worldgen/` — Interactive terrain generation sandbox with Whittaker-style continent/island/highland presets, elevation bands (Water/Lowland/Hills/Mountains), slope, soil fertility/hold, roughness, and precomputed downhill routing fields. Nine overlay modes (Height, Slope, ElevBand, DistOcean, DistWater, SoilFertility, SoilHold, Roughness). Live parameter tweaking for WaterLevel, NoiseScale, ContinentThreshold, RidgeStrength.
- `sandboxes/weather/` — Atmospheric weather system layered on worldgen terrain. Static bake: latitude temperature, 3-cell wind model, orographic precipitation sweeps, storminess, rain shadow. Atmosphere simulation (`atmosphere.h/cpp`): coarse-grid (4× tile scale) atmospheric state with T/q/p/wind/cloud/precip_rate/storminess per cell, latent heat from condensation, solar forcing, temperature relaxation toward T_ref, east-west periodic boundary advection, thermal turbulence with convective gusts; auto-starts after bake. Dynamic simulation: precipitation as rain (capped at 1.0)/snow, snowmelt, infiltration, evaporation, surface water downhill runoff, terrain humidity feedback; reads from atmosphere when active. Telemetry: `AtmosphereStats` (T/q/cloud/precip/wind/pressure/storm ranges, raining_cells, cloudy_cells), live legend panel, console diagnostics every 2 sim-days. Overlay keys 1-9, 0, -, = covering Temperature through SnowDepth; Moisture shows atmospheric humidity (q), RainShadow traces current wind field. Controls: T=play/pause, R=reset dynamics, Numpad+/-=speed.
- Physical invariant enforcement in `sandboxes/weather/` atmosphere simulation: Stefan-Boltzmann T^4 radiative cooling, water mass conservation monitoring with drift correction (>1% threshold), Clausius-Clapeyron q_sat hard clamp, wind speed cap (MAX_WIND_SPEED=6), explicit Laplacian heat and moisture diffusion with periodic wrap, convective instability release, soil-moisture-limited evaporation, CFL/Courant advection limiter, and surface water overflow-to-soil rerouting. `AtmosphereStats` extended with `total_atmo_water`, `max_courant`, `water_correction` fields for live invariant diagnostics.
- `sandboxes/weather/` atmosphere simulation tuning and bug fixes: restructured tick order so solar temperature forcing runs AFTER advection (critical — prevented advection from overwriting forced equatorial temperatures with stale upwind values, collapsing the latitude gradient); added partial temperature advection (15% local retention) to preserve latitude gradient against numerical diffusion; added minimum equatorial Coriolis (0.04 floor) to deflect meridional flow into zonal trade winds; retuned physics constants (K_SOLAR=8.0, K_RELAX_T=0.05 nonlinear, K_DAMP=0.05, ADVECT_SPEED=0.5) for climate stability; tightened water drift correction threshold to 0.1%.
- `sandboxes/weather/` headless diagnostic mode: `sandbox_weather --headless N` runs N sim-days without an SDL2 window and prints per-2-day latitude-band temperature and wind diagnostics, then warns if temperature spread, band spread, or wind speed fall below stability thresholds. Enables atmosphere simulation verification in headless/CI environments.
- `sandboxes/weather/` `AtmosphereStats` extended with `T_band[3]`, `wind_band[3]`, `q_band[3]` arrays (north polar / equatorial / south polar thirds) for latitude-band diagnostics in both headless output and console diagnostic dumps.
- `sandboxes/veggen/` — Vegetation simulation across 8 biome presets (namespace `veggen`). Two-layer moisture model (surface_water + root_moisture), hydrology pipeline (rain→surface→infiltration→root, evaporation reduced by canopy, runoff to downhill neighbors), canopy cover feedback. Five death causes (Drought, Cold, Heat, Competition, Senescence). Overlays: None, RootMoisture, Temperature, Health, Elevation, SurfaceWater, CanopyCover (keys 1-7). Headless modes: `--auto` (seasonal BMP screenshots), `--metrics` (CSV export). Metrics snapshots every 30 sim-days.
- `sandboxes/spritetest/` — Sprite rendering test harness; shares veggen source files directly.
- Docs-maintainer skill at `.claude/skills/docs-maintainer/SKILL.md` with tiered file update procedure.
- System design docs: `docs/systems/TERRAIN_GENERATION.md` (worldgen sandbox pipeline), `docs/systems/WEATHER_SYSTEM.md` (weather sandbox all three phases), `docs/systems/PLANT_GENERATION.md` (veggen simulation and plant sprites).

### Changed
- `sandboxes/worldgen/` renamed generation types: `world_gen.h/cpp` → `terrain_gen.h/cpp`, `World` struct → `Terrain`, `generate_world` → `generate_terrain`, `WorldStats` → `TerrainStats`. Weather sandbox synced to the same rename (`world_gen.h/cpp` → `terrain_gen.h/cpp`). These renames bring naming in line with what the sandbox actually generates (terrain, not a full game world).
- `sandboxes/worldgen/` overlay legend panels now show a short info description line beneath the gradient bar or swatches, explaining what the field represents and how it was computed (applies to all 11 overlays).
- `sandboxes/worldgen/` camera now starts fitted to show the full world at launch via `Camera::fit_world` — no need to zoom out manually.
- `sandboxes/worldgen/` terrain pipeline overhauled: domain warping added to height field generation (two Perlin displacement layers warp input coordinates up to 20 tiles for organic landforms); ridge field merged into unified height so mountains affect hydrology, slope, distance fields, and band classification; Euclidean Distance Transform (Felzenszwalb separable parabola-envelope) replaces BFS Manhattan distances for `dist_ocean` and `dist_water`; aspect field added (`atan2` slope direction, [-π, π]); roughness changed from ridged-noise+slope blend to 3×3 neighborhood height standard deviation; downhill routing upgraded to 8-directional D8 with lake outflow spill points; flow accumulation implemented replacing the previous stub (river_flow was 0 in all tiles). Two new overlay modes added: Aspect (key 0) and RiverFlow (key -). `TerrainTile` gained `aspect`, `downhill_x`, `downhill_y` fields. `GenerationTimings` now has `ridge_ms`, `unify_ms`, `downhill_ms`, `river_ms` stages (replacing old `elevation_ms`).
- `sandboxes/weather/` terrain generation synced with worldgen pipeline changes (domain warping, unified height, aspect, D8 downhill, flow accumulation, EDT distance fields).
- `sandboxes/weather/` temperature range widened: `T_ref` formula changed from `-10 + solar * 35` to `-15 + solar_seasonal * 45` with stronger seasonal latitude modulation (`* 12` vs `* 8`). This enables realistic heat waves (equatorial T > 30°C in summer) and deeper polar cold, giving the atmosphere a wider and more realistic temperature range.
- `sandboxes/weather/` storminess formula rewritten from additive (which saturated ~90% of cells, making storminess uniform and meaningless) to multiplicative with convergence threshold: `humidity_ratio * instability^3 * (1 + cloud*0.5)` where instability requires wind convergence > 0.2. Produces realistic ~15-19% severe storm cell coverage concentrated in meteorologically active zones.
- `sandboxes/weather/` headless diagnostic final summary now reports cumulative peak extremes tracked across the entire simulation run — peak T range, peak heat wave / deep freeze / severe storm / gale / heavy rain cell counts, and peak per-cell precipitation intensity — rather than a snapshot of the final timestep only.
- `sandboxes/weather/` precipitation is now intermittent: smooth bilinear noise (10-cell + 4-cell scales, drifting with wind, evolving every ~12 sim-hours) gates rain delivery — only ~45% of cloudy cells rain at any given time, creating coherent weather fronts rather than uniform drizzle across all cloudy cells. Rain intensity is 2.5× higher when triggered to preserve total moisture budget. Orographic precipitation (mountain uplift) remains continuous. Headless per-2-day diagnostics now report cloudy cell count alongside raining cell count to monitor spatial patchiness.
- `sandboxes/weather/` precipitation model replaced: noise-gated intermittency removed in favour of a physics-based cloud lifecycle. Noisy condensation thresholds cause cloud to thicken; precipitation falls when cloud exceeds CLOUD_PRECIP_THRESHOLD (0.12) at CLOUD_PRECIP_RATE (1.2), then cloud dissipates via CLOUD_EVAP_BASE (0.15), CLOUD_EVAP_DRY (0.40), and pressure-driven CLOUD_SUBSIDE_RATE (0.50). Condensation noise uses cubic Hermite temporal crossfade between timesteps to prevent global pattern teleport; a second decorrelated noise layer (spatial offset +1000) prevents it from reinforcing the first. `hash01()` changed to `uint32_t` parameters to fix signed integer overflow UB. OCEAN_EVAP_RATE reduced from 0.35 → 0.15; LAND_EVAP_RATE = 0.05 added. T_ref solar multiplier increased from 45 → 48.
- `sandboxes/weather/` Precipitation and Storminess overlays now use `AtmosphereState::sample_nearest()` (new method) instead of bilinear `sample()`, producing sharp shower boundaries that match the underlying coarse-grid cell structure rather than smeared gradients.
- `docs/systems/` overhauled: removed five stale/unimplemented system docs (PLANETARY_MODEL.md, NUTRIENT_REGENERATION.md, NEURAL_EVOLUTION.md, WORLD_GENERATION.md, SPRITE_GENERATION.md); replaced with three docs covering sandbox-prototyped systems (TERRAIN_GENERATION.md, WEATHER_SYSTEM.md, PLANT_GENERATION.md).
- `docs-maintainer` skill Tier 3 updated to reference new system doc file names.
- Sandbox UI skill at `.claude/skills/sandbox-ui/SKILL.md` — layout constants, spacing rules, color conventions, and performance thresholds for SDL2 sandbox UI changes.
